Course Description

This course is mainly designed for students who intend on participating in computing contests, such as University of Waterloo's Canadian Computing Competition (CCC). The contents of this course will help aid the students prepare for the upcoming contests. The first half of the course is mostly a review of the topics covered in the Introduction to Programming courses (e.g., branching statements, loops, recursion, simple linear data structures like arrays and lists, etc.); contest-like problems that utilize these topics are also covered. The second half of this course focuses on more advanced topics of computing that also come up in programming contests, such as stacks, queues, trees, and simple sorting algorithms. Overall, while the course is geared towards students who intend on joining computing contests, it is also open to anyone with basic programming background who intends on expanding programming knowledge, improving problem-solving skills, and/or solving programming challenges. Students who complete this course will gain eligibility to take on the second level of the CCC course.

Course Prerequisites: Introduction to Programming (either in Python, Java, or C++)

Note: Please contact your instructor if you have not completed the prerequisites for this course.

If you took my class previously, there will be some slight changes to how you submit homeworks. For each homework you submit, you will do the following steps.
  • Step 0: This step may only need to be done once throughout the course. Setup Classkick. See below section for instructions regarding Classkick.
  • Step 1: First create a folder with the following filename: firstname-lastname-homework-X, where X is the HW number. So for example, I will name my folder as Gian-Alix-homework-01 for my first homework submission.
  • Step 2: Place all files that you would like to submit to this folder. NEW: Your solution to each problem should be in a separate file. So after compressing, your filename should be something like Gian-Alix-homework-01.zip
  • Step 4: Go to Classkick. Take a screenshot of your codes and paste them in the homework submission page. It also has the link for submitting the compressed folder; please upload the compressed folder to that link.
  • Step 5: Feel free to upload as many times as you wish. Previously uploaded files are overwritten. Only the last submission will be graded.

Feel free to ask me questions about homework submission (especially if you're not sure how or where to find the /PycharmProjects directory) and I will be able to help you.

Grading Scheme

For each question/problem, it is worth 5 points. Here is a grading scheme for it:

  • 5 points - If the Online Judge (OJ) deems the answer is correct (passes ALL test cases given a reasonable amount of time). I won't write feedback if you got 5 points - that's how good you are! :) .
  • 4 points - Minor issues. Maybe 1 or 2 failed test cases. But that's ok, I tell you this.
  • 3 points - Around half of the test cases passed, and the rest fail.
  • 2 points - Major issues. Very few test cases passed (possibly 1?)
  • 1 point - Wrong Answer (WA). No test case passed. Any kind of error (syntax, runtime, etc.) caught by the online compiler.
  • 0 point - No attempt.

Majority of the homework questions are actually contest problems and an OJ is grading them. Unfortunately, it's part of the contest that the judge won't tell you what are the set of inputs that fail -- sometimes, it doesn't even tell you how many fail!

Another advice that I have when submitting solutions to these problems is that the OJ is very picky! Don't output text that is not required. So for example, don't output "Please input an integer: " when prompting for an input (unless specified otherwise by the problem!). You also have to be careful with capitalization/lower-case (output to the console must be case-senstive). Don't add extra or remove any characters such as punctuations like spaces, periods, etc. The expected output of the OJ must match your actual output. Code with efficient runtime is also expected in computing competitions, but we'll talk about it later down the line.

I was lenient with this grading in the first few homeworks, but will become more strict down the line as I will be mostly relying on the OJ when grading. Not all problems however are being marked with an OJ (only those that can be found in DMOJ). To help you with testing your code with the online judge for HW 2, I highly recommend (not required) to sign up for an account with DMOJ and then submit your solutions here: HW 2 - Q1 (Icon Scaling), HW 2 - Q2 (Who Has Seen The Wind?), HW 2 - Q3 (Up and Down). Note that submitting in the DMOJ doesn't mean that you already submit the HW for this class; you still need to follow the proper proecudre for submitting the homework.
