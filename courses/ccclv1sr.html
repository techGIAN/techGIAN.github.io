Course Description

This course is mainly designed for students who intend on participating in computing contests, such as University of Waterloo's Canadian Computing Competition (CCC). The contents of this course will help aid the students tackle challenging computing problems from various computing contests. Compared with CCC Level 1 Jr, this course covers more advanced concepts (including but not limited to disjoint sets, graph search algorithms, advancing sorting algorithms, minimum spanning trees, introductory dynamic programming, counting, number theory, combinatorics, heaps, and segment trees); and discussions on these topics are complemented by examining computing contest problems. Overall, while the course is geared towards students who intend on writing computing contests, it is also open to anyone with basic programming background who intends on expanding programming knowledge, improving problem-solving skills, and/or solving programming challenges. Students who complete this course will gain eligibility to take on the second level of the CCC course.

Part 1 (Concepts)

You are given a one page on Classkick (more info on Classkick below) to write your answers to the conceptual questions. Show all work as needed. It must fit that one page; otherwise, your answers are too long and you may need to cut down. Another option is to type your answers in a Word Processor, and then take a screenshot and paste there. Another alternative is to write your answers on a piece of paper and then attach it there. Regardless of your method, you still need to be able to fit your work in that single page.

Part 2 (Contest Problem)

Read the contest problem carefully, including the input and output specifications, and analyze as well the sample input and output. Then code a solution for the problem being asked (either in Python, Java, or C++). See below for instructions for how to submit your solutions:
  • Step 0: This step may only need to be done once throughout the course. Setup Classkick. See below section for instructions regarding Classkick.
  • Step 1: First create a folder with the following filename: firstname-lastname-homework-X-Sr, where X is the HW number. So for example, I will name my folder as Gian-Alix-homework-01-Sr for my first homework submission. If you took my CCC class last Fall 2024 term, this folder name format is not that different, with the exception of the "Sr" suffix in the filename.
  • Step 2: Place all files that you would like to submit to this folder. Your solution to each problem should be in a separate file. So if there are three contest problems in the HW for example, then the folder should contain three files, with each file containing the solution to each homework question.
  • Step 3: Once all files that you would like to submit are now in the folder, compress it using any of the zipping softwares. For Windows such as: WinRar, 7zip, or WinZip (no need to install if your machine already has one). For Mac, a compression utility software is already built-in so no need to install (simply right click on the folder, then click on Compress). So after compressing, your filename should be something like Gian-Alix-homework-01-Sr.zip
  • Step 4: Go to Classkick. Take a screenshot of your codes and paste them in the homework submission page. It also has the link for submitting the compressed folder; please upload the compressed folder to that link.
  • Step 5: Feel free to upload as many times as you wish. Previously uploaded files are overwritten. Only the last submission will be graded.

Feel free to ask me questions about homework submission (especially if you're not sure how or where to find the project directory) and I will be able to help you.

Grading Scheme

Your grade for Part 1 will be added with Part 2. For Part 1, the highest possible score can always vary, depending on the Homework (grading scheme taken up during Homework Takeup). For Part 2, the highest possible score for each problem is always out of 5:

  • 5 points - If the Online Judge (OJ) deems the answer is correct (passes ALL test cases given a reasonable amount of time). I won't write feedback if you got 5 points - that's how good you are! :) .
  • 4 points - Minor issues. Maybe 1 or 2 failed test cases. But that's ok, I tell you this.
  • 3 points - Around half of the test cases passed, and the rest fail.
  • 2 points - Major issues. Very few test cases passed (possibly 1?)
  • 1 point - Wrong Answer (WA). No test case passed. Any kind of error (syntax, runtime, etc.) caught by the online compiler.
  • 0 point - No attempt.

Majority of the homework questions are actually contest problems and an OJ is grading them. Unfortunately, it's part of the contest that the judge won't tell you what are the set of inputs that fail -- sometimes, it doesn't even tell you how many fail!

Another advice that I have when submitting solutions to these problems is that the OJ is very picky! Don't output text that is not required. So for example, don't print "Please input an integer: " when prompting for an input (unless specified otherwise by the problem!). You also have to be careful with capitalization/lower-case (output to the console must be case-senstive). Don't add extra or remove any characters such as punctuations like spaces, periods, etc. The expected output of the OJ must match your actual output. Code with efficient runtime is also expected in computing competitions.

Total Score

For each Homework, I tally your total scores. If it is 75% (or above) in terms of percentage, then you will be marked "P" in the report card for that Homework. If it is below that, then it is "I". If no submission attempt made, then it will be blank.If you need to resubmit to fix corrections (up to Attempt 2) or submitting a missed HW, then the filename should then have a format be Gian-Alix-homework-01-Sr-resubmitted.zip. When resubmitting HW, make sure to use the Raise Hand functionality of Classkick; otherwise, they will not be seen and thus not be graded.
